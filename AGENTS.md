# AGENTS.md

## Stack
- Astro 6 (static, no SSR adapter) with Vue 3 + motion-v, TypeScript 6.

## Package manager
- Use `pnpm`. `devEngines.packageManager` in `package.json` errors on other package managers.
- `package-lock.json` is a stub (only one empty entry); only `pnpm-lock.yaml` is real.
- `pnpm-workspace.yaml` has no workspaces — only pnpm settings:
  - `allowBuilds` blocks `esbuild` and `sharp` postinstall scripts.
  - `@vue/*` 3.5.38 and `vue@3.5.38` are pinned via `minimumReleaseAgeExclude` (lockfile resolves to 3.5.39 via the override; don't be surprised by the version skew).

## Commands
- `pnpm install` — install deps.
- `pnpm add [package]` — add a dependency (use this, not `npm install <pkg>`).
- `pnpm dev` — `astro dev --host 0.0.0.0 --port 4321` (binds all interfaces; port is hardcoded).
- `pnpm build` — `astro build` (outputs to `dist/`).
- `pnpm preview` — `astro preview`.
- `pnpm exec astro check` — typecheck + Astro diagnostics. There is no `lint`/`test`/`typecheck` script in `package.json`; `astro check` is the only built-in verification step.
- `pnpm exec prettier --write .` — format. Configured in `.prettierrc.json` with `prettier-plugin-astro` and an `astro` parser override.

## TypeScript
- `tsconfig.json` extends `astro/tsconfigs/base` and `@vue/tsconfig/tsconfig.json`.
- LSP plugins enabled: `@astrojs/ts-plugin`, `@vue/typescript-plugin`.
- Includes `.astro/types.d.ts` (regenerated by `astro sync`/dev). Excludes `dist`.

## Generated / build artifacts (do not edit by hand)
- `.astro/` — generated types (`content.d.ts`, `types.d.ts`, `settings.json`).
- `dist/` — build output. Listed in `.gitignore` but present in the tree; treat as regenerable via `pnpm build`.
- `node_modules/`, `.pnpm-store/` — local/cached.

## Conventions
- `astro.config.mjs` registers only the Vue integration. No content collections, no MDX, no SSR adapter — site is a single static page.
- `.helix/languages.toml` is the repo's Helix editor config (Vue LSP plugin, Prettier auto-format for `typescript`/`vue`/`astro`); preserve it as-is.
- No CI, no pre-commit hooks, no test framework configured.
