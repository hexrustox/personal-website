# AGENTS.md

## Stack

- Astro 6 (static, no SSR adapter) with Vue 3 + motion-v, TypeScript 6.

## src/ layout

- `src/components/` — `.vue` SFCs.
- `src/pages/` — `.astro` pages (`_app.ts` is the Vue app entrypoint, referenced from `astro.config.mjs`; empty by design).
- `src/lib/` — `.ts` files.
- `src/styles/` — global CSS.

## Package manager

- Use `pnpm`. `devEngines.packageManager` in `package.json` errors on other package managers.
- `package-lock.json` is a stub (only one empty entry); only `pnpm-lock.yaml` is real.
- `pnpm-workspace.yaml` has no workspaces — only pnpm settings:
  - `allowBuilds` blocks `esbuild` and `sharp` postinstall scripts.
  - `@vue/*` 3.5.38 and `vue@3.5.38` are pinned via `minimumReleaseAgeExclude` (lockfile resolves to 3.5.39 via the override; don't be surprised by the version skew).

## Commands

- `pnpm install` — install deps.
- `pnpm add [package]` — add a dependency (use this, not `npm install <pkg>`).
- `pnpm dev` — `astro dev` (binds `0.0.0.0:3000`; see `astro.config.mjs`).
- `pnpm build` — `astro build` (outputs to `dist/`).
- `pnpm preview` — `astro preview`.
- `pnpm check` — `vue-tsc && astro check`. There is no `lint`/`test`/`typecheck` script in `package.json`; this is the only built-in verification step.
- `pnpm fmt` — `prettier -w src/`. `.prettierrc.json` enables `prettier-plugin-astro` (with an `astro` parser override) and `@trivago/prettier-plugin-sort-imports`.

## Path aliases

- `@lib/*`, `@components/*`, `@styles/*` are resolved at the type level only, via `tsconfig.json` (`compilerOptions.paths`).
- `unplugin-vue-components` (in `astro.config.mjs`) auto-registers every `.vue` file in `src/components/` plus `Motion` / `AnimatePresence` from `motion-v` into `components.d.ts` — generated, do not edit.

## TypeScript

- `tsconfig.json` extends `astro/tsconfigs/base` and `@vue/tsconfig/tsconfig.json`.
- LSP plugins enabled: `@astrojs/ts-plugin`, `@vue/typescript-plugin`.
- Includes `.astro/types.d.ts` (regenerated by `astro sync`/dev). Excludes `dist`.

## CSS

- Naming convention: BEM — `.block`, `.block__element`, `.block--modifier`. Use `__` for elements and `--` for modifiers. Block names are kebab-case (e.g. `.about-me`, not `.aboutMe`).
- Scope component styles with `<style scoped>` in `.vue`/`.astro` files (Astro default). Global CSS lives in `src/styles/`.
- Theme tokens are CSS custom properties on `:root` in `src/styles/global.css` (e.g. `--bg`, `--text`, `--accent`, `--font-body`, `--font-display`); reference them via `var(--token)` rather than hardcoding values.

## Browser automation (playwright-cli)

- A `playwright-cli` skill ships at `.agents/skills/playwright-cli/SKILL.md` — load it for the full command reference. Bare essentials: `playwright-cli open [url]`, `playwright-cli snapshot`, `playwright-cli click <ref>`, `playwright-cli close`.
- **Only Firefox is available in this dev environment** (the Nix flake builds the shell on `mcr.microsoft.com/playwright:v1.61.0-noble` with only the firefox browser installed). Always pass `--browser firefox` to `playwright-cli open`, e.g. `playwright-cli open http://localhost:3000 --browser firefox`. Chrome / webkit / msedge will fail.
- Snapshots and console logs land in `.playwright-cli/` (gitignored). `test-results/` is also playwright-cli scratch — both are safe to delete.

## Dev environment

- Dev shell is provided by a Nix flake (`flake.nix`) using `nix-capsule`; `.envrc` is set up for `direnv` (`use flake .`). If `direnv` is not active, enter the shell with `nix develop` (or `nix run .#`) manually.
- The shell runs inside a container with Node, pnpm, the language servers, and Playwright + Firefox preinstalled. Port 3000 is forwarded (`-p=3000:3000`) for `astro dev`.

## Generated / build artifacts (do not edit by hand)

- `.astro/` — generated types (`content.d.ts`, `types.d.ts`, `settings.json`).
- `components.d.ts` — generated by `unplugin-vue-components`; auto-registers every `.vue` in `src/components/` and motion-v's `Motion` / `AnimatePresence`.
- `dist/` — build output. Listed in `.gitignore` but present in the tree; treat as regenerable via `pnpm build`.
- `node_modules/`, `.pnpm-store/`, `.direnv/`, `.ncap-cache/`, `.playwright-cli/`, `test-results/` — local/cached.

## Conventions

- `astro.config.mjs` registers only the Vue integration. No content collections, no MDX, no SSR adapter — site is a single static page.
- `.helix/languages.toml` is the repo's Helix editor config (Vue LSP plugin, Prettier auto-format for `typescript`/`vue`/`astro`); preserve it as-is.
- No CI, no pre-commit hooks, no test framework configured.
