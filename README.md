# personal-website

> Source for [ckhgy.com](https://ckhgy.com) — my personal site and portfolio.

## Tech stack

- [Astro 7](https://astro.build) (static, no SSR adapter)
- [Vue 3](https://vuejs.org) with [`motion-v`](https://motion.dev) for animation
- [TypeScript 6](https://www.typescriptlang.org)
- [`unplugin-vue-components`](https://github.com/unplugin/unplugin-vue-components) — auto-imports every `.vue` in `src/components/` plus `Motion` / `AnimatePresence` from `motion-v`
- [`@astrojs/sitemap`](https://docs.astro.build/en/guides/integrations-guide/sitemap/) for sitemap generation
- [Playwright](https://playwright.dev) + Firefox (browser automation, see [`.agents/skills/playwright-cli/`](.agents/skills/playwright-cli/SKILL.md))
- [Caddy](https://caddyserver.com) for serving the production build

## Project layout

```
.
├── astro.config.mjs     # Astro + Vue + sitemap + unplugin-vue-components
├── Caddyfile            # Production server (dist/ on :3001)
├── flake.nix            # Nix dev shell
├── public/              # Static assets (favicon, logo, fonts)
└── src/
    ├── components/      # .vue SFCs (auto-registered)
    │                    #   Hero, Nav, AboutMe, Skills, Projects,
    │                    #   Timeline, Contact, Link, Icon, ...
    │                    #   Animation primitives: FadeOnKey, Reveal,
    │                    #   RevealGroup, TextSplit, SectionHeading,
    │                    #   YearLabel
    ├── data/            # Page content (TypeScript):
    │                    #   meta, hero, about, skills, contact,
    │                    #   projects, timeline
    ├── layouts/         # .astro layouts
    ├── lib/             # Shared utilities (motion helpers, hooks)
    ├── pages/           # .astro pages
    │                    #   index.astro, 404.astro,
    │                    #   _app.ts (Vue app entrypoint),
    │                    #   robots.txt.ts
    └── styles/          # Global CSS (theme tokens on :root)
```

Content lives in `src/data/*.ts`; components in `src/components/`. Edit the data, not the components, for copy changes.

## Development

### Prerequisites

- **pnpm 11** — enforced via `devEngines.packageManager` in `package.json`. Other package managers will error.
- **Node** — provided by the Nix shell, or install separately.
- **direnv** (optional) — for automatic activation of the Nix flake shell.

### Run the dev server

```sh
pnpm install
pnpm dev
```

The dev server binds `0.0.0.0:3000` (see `astro.config.mjs`).

### Scripts

| Command       | What it does                                       |
| ------------- | -------------------------------------------------- |
| `pnpm dev`    | `astro dev` (binds `0.0.0.0:3000`)                 |
| `pnpm build`  | `astro build` (outputs to `dist/`)                 |
| `pnpm preview`| `astro preview`                                    |
| `pnpm check`  | `vue-tsc && astro check` — the only built-in check |
| `pnpm fmt`    | `prettier -w src/`                                 |

There is no `lint`, `test`, or `typecheck` script. `pnpm check` is the only built-in verification step.

### Editing content

`src/data/` is the single source of truth for site copy:

- `meta.ts` — name, role, location, tagline, availability
- `hero.ts`, `about.ts`, `skills.ts`, `projects.ts`, `timeline.ts`, `contact.ts`

Add a new section by creating `src/data/<section>.ts`, dropping a new `.vue` SFC in `src/components/`, and referencing it from `src/pages/index.astro`. Auto-registration means no manual import is needed for the component.

### Code conventions

- **BEM** everywhere: `.block`, `.block__element`, `.block--modifier`. Kebab-case block names (e.g. `.about-me`).
- Scope component styles with `<style scoped>` in `.vue` / `.astro` files. Global CSS lives in `src/styles/`.
- Theme tokens are CSS custom properties on `:root` in `src/styles/global.css` (e.g. `--bg`, `--text`, `--accent`, `--font-body`, `--font-display`). Reference them via `var(--token)`.
- TypeScript path aliases: `@components/*`, `@styles/*` (resolved at the type level via `tsconfig.json`).
- Imports are sorted by `@trivago/prettier-plugin-sort-imports`. Run `pnpm fmt` before committing.
- `components.d.ts` is generated by `unplugin-vue-components` — do not edit it by hand.

## Build & deploy

```sh
pnpm build
```

Outputs a static site to `dist/`. The included `Caddyfile` serves it on `:3001` with:

- HSTS (`max-age=63072000; includeSubDomains; preload`) and security headers (`X-Content-Type-Options`, `X-Frame-Options DENY`, `Referrer-Policy`)
- `Cache-Control: public, max-age=86400, immutable` for `/_astro/*` and `/fonts/*`
- `Cache-Control: public, max-age=3600, must-revalidate` for HTML
- `encode zstd gzip` compression
- Custom 404 fallback to `404.html` (`hide 404.html` from the listing)
